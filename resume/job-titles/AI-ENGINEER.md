## What this role is

An **AI Engineer** builds and operates **production systems powered by foundation models** — LLM-integrated apps, RAG/agent workflows, speech/vision pipelines, and the APIs, data paths, and observability around them. In 2025–2026 hiring, this is less “train a novel architecture” and more **integrate, evaluate, ship, and tune** models under real latency, cost, and reliability constraints.

The role overlaps heavily with **Forward Deployed Engineer** (customer-facing delivery), **Data Scientist** (modeling/evaluation), and **Public Technologist** (mission-driven AI) — same ACLU work can be tagged for multiple roles with different emphasis.

**Distinct emphasis vs siblings:** product-facing **GenAI integration** (RAG, agents, auto-labeling, inference optimization) over pure policy narrative (Public Technologist) or statistical storytelling alone (Data Scientist).

## What employers look for (2025–2026 market)

| Priority | Signals |
|----------|---------|
| **LLM / GenAI in production** | API integration, RAG, agent orchestration, eval/guardrails — often cited more than fine-tuning alone |
| **Multi-modal & NLP pipelines** | ASR → text → classification; unstructured video/audio/text |
| **Production Python + APIs** | FastAPI, Docker, batch/online inference, CI/CD |
| **Cloud & ops** | AWS/GCP, containers, monitoring, cost/latency tradeoffs |
| **Classical ML literacy** | Supports GenAI features — classification, threshold tuning, feature pipelines |

**Typical stack:** Python, PyTorch/Transformers, FastAPI, PostgreSQL, Whisper/speech, OpenAI or open-weight LLMs, Docker, React or internal UI, vector DB when RAG-heavy.
